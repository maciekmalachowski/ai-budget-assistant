# AI Budget Assistant — Phase 1: Foundations Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a runnable Next.js 15 + TypeScript app with Tailwind v4, shadcn/ui prerequisites, Vitest, and ESLint — plus the `.env.example` and a passing smoke test — so later phases have a solid, tested base to build on.

**Architecture:** A single Next.js (App Router) project at the repo root. Reads will favor server components in later phases; this phase only establishes tooling and a minimal app shell. No database, auth, or AI yet — and no secrets required.

**Tech Stack:** Next.js 15, React 19, TypeScript 5, Tailwind CSS v4 (`@tailwindcss/postcss`), shadcn/ui (new-york / neutral), Vitest + Testing Library + jsdom, ESLint 9 (flat config via `eslint-config-next`).

> **Note on versions:** Exact patch versions below are known-good as of 2026-05; if `npm install` resolves a newer compatible minor, that's fine. The build/lint/test verification steps at the end are the source of truth — if they pass, the scaffold is correct.

> **Phase context:** This is Phase 1 of 7 (see `docs/superpowers/specs/2026-05-31-ai-budget-assistant-design.md` and the phase map). Subsequent phases (domain logic, data model, AI, API/auth, UI, deploy) get their own plans once this one is executed.

```ts
import { describe, it, expect } from "vitest";
import { cn } from "@/lib/utils";

describe("cn", () => {
  it("merges conflicting tailwind classes, keeping the last", () => {
    expect(cn("p-2", "p-4")).toBe("p-4");
  });

  it("drops falsy values and joins the rest", () => {
    expect(cn("a", false && "b", undefined, "c")).toBe("a c");
  });
});
```
